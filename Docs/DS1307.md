# Librería DS1307 de solo lectura

## 1. Descripción

La librería `DS1307` permite leer la fecha y hora almacenadas en un reloj de tiempo real **DS1307** mediante el bus I2C.

![DS1307 - Imagen](../Docs/img/ds1307.jfif)

Esta versión es exclusivamente de lectura:

- No configura la hora.
- No configura la fecha.
- No modifica el bit CH.
- No modifica el registro de control.
- No escribe en la memoria RAM del DS1307.

La librería utiliza `I2C.h` como capa inferior de comunicación.

---

## 2. Archivos

```text
DS1307.h
DS1307.c
```

### Dependencias

```text
I2C.h
I2C.c
```

La aplicación también puede utilizar:

```text
LCD_I2C.h
LCD_I2C.c
```

para mostrar los datos, pero el LCD no es una dependencia directa de la librería DS1307.

---

## 3. Características

- Dirección I2C fija de 7 bits: `0x68`.
- Lectura de segundos, minutos y horas.
- Lectura del día de la semana.
- Lectura del día del mes.
- Lectura del mes y año.
- Conversión automática de BCD a decimal.
- Compatibilidad con modo de 12 y 24 horas.
- Detección AM/PM.
- Detección del bit CH.
- Validación básica de rangos.
- Lectura cruda de registros.
- Lectura de bloques de registros.

---

## 4. Conexiones

```text
PIC18F57Q43              DS1307
-----------------------------------------
RC3 / SCL      --------> SCL
RC4 / SDA      <-------> SDA
5 V            --------> VCC
GND            --------> GND
Batería 3 V    --------> VBAT
```

El cristal de 32.768 kHz debe estar conectado entre X1 y X2 si el módulo no lo incluye.

El DS1307 trabaja normalmente con VCC de 5 V. La batería mantiene el reloj funcionando cuando se retira la alimentación principal.

Cuando el dispositivo funciona solamente con batería, conserva la hora, pero no permite el acceso normal al bus hasta recuperar VCC.

---

## 5. Dirección I2C

```c
#define DS1307_I2C_ADDRESS 0x68u
```

La librería I2C transforma automáticamente esta dirección:

```text
Escritura: 0xD0
Lectura:   0xD1
```

La aplicación no debe utilizar directamente `0xD0` ni `0xD1`.

---

## 6. Mapa de registros

| Registro | Nombre | Contenido |
|---:|---|---|
| `0x00` | Segundos | Segundos en BCD y bit CH |
| `0x01` | Minutos | Minutos en BCD |
| `0x02` | Horas | Hora en modo 12 o 24 horas |
| `0x03` | Día | Día de la semana |
| `0x04` | Fecha | Día del mes |
| `0x05` | Mes | Mes |
| `0x06` | Año | Últimos dos dígitos del año |
| `0x07` | Control | Configuración de SQW/OUT |
| `0x08–0x3F` | RAM | Memoria RAM respaldada por batería |

La librería de fecha y hora lee normalmente los registros `0x00` a `0x06`.

---

## 7. Formato BCD

El DS1307 almacena la mayoría de los valores en BCD.

Ejemplo:

```text
0x34 BCD = 34 decimal
```

No corresponde a:

```text
0x34 hexadecimal = 52 decimal
```

La conversión se realiza internamente en la librería.

Ejemplo de resultado:

```c
fechaHora.minutes = 34u;
```

aunque el registro del DS1307 contenga `0x34`.

## 8. Estructura `DS1307_DateTime`

```c
typedef struct
{
    uint8_t seconds;
    uint8_t minutes;
    uint8_t hours;

    uint8_t weekday;
    uint8_t date;
    uint8_t month;
    uint16_t year;

    uint8_t clock_running;
    uint8_t mode_12h;
    uint8_t is_pm;
    uint8_t data_valid;

} DS1307_DateTime;
```

### Campos

| Campo | Descripción |
|---|---|
| `seconds` | Segundos de 0 a 59. |
| `minutes` | Minutos de 0 a 59. |
| `hours` | Horas en decimal. |
| `weekday` | Día de la semana de 1 a 7. |
| `date` | Día del mes de 1 a 31. |
| `month` | Mes de 1 a 12. |
| `year` | Año completo, por ejemplo 2026. |
| `clock_running` | Indica el estado del bit CH. |
| `mode_12h` | Indica si el registro usa formato de 12 horas. |
| `is_pm` | Indica PM en modo de 12 horas. |
| `data_valid` | Indica si los registros contienen valores válidos. |

---

## 9. Indicador `clock_running`

El bit 7 del registro de segundos es el bit `CH`.

```text
CH = 0 → oscilador habilitado
CH = 1 → oscilador detenido
```

La librería entrega:

```c
fechaHora.clock_running
```

| Valor | Significado |
|---:|---|
| `1` | El reloj está funcionando. |
| `0` | El reloj está detenido. |

Como la librería es de solo lectura, no puede cambiar CH de 1 a 0.

---

## 10. Modos de hora

### Modo de 24 horas

```c
fechaHora.mode_12h == 0u
```

El campo `hours` contiene valores entre 0 y 23.

### Modo de 12 horas

```c
fechaHora.mode_12h == 1u
```

El campo `hours` contiene valores entre 1 y 12.

La indicación AM/PM se obtiene mediante:

```c
fechaHora.is_pm
```

| Valor | Significado |
|---:|---|
| `0` | AM |
| `1` | PM |

## 11. Funciones públicas

### 11.1 `DS1307_ReadRegisters`

```c
I2C_Status DS1307_ReadRegisters(
    uint8_t start_register,
    uint8_t *data,
    uint8_t length
);
```

Lee uno o varios registros consecutivos.

#### Parámetros

| Parámetro | Descripción |
|---|---|
| `start_register` | Primer registro que será leído. |
| `data` | Arreglo de destino. |
| `length` | Cantidad de bytes que se leerán. |

Ejemplo:

```c
uint8_t registros[3];
I2C_Status estado;

estado = DS1307_ReadRegisters(
    DS1307_REG_SECONDS,
    registros,
    3u
);
```

Resultado:

```text
registros[0] → segundos crudos
registros[1] → minutos crudos
registros[2] → horas crudas
```

### Aclaración sobre la escritura del puntero

Para seleccionar el registro inicial, la función ejecuta internamente:

```c
I2C1_WriteSingleByte(
    DS1307_I2C_ADDRESS,
    start_register
);
```

Esta operación no cambia la hora ni la fecha. Solamente coloca el puntero interno del DS1307 en el registro desde el cual se realizará la lectura.

---

### 11.2 `DS1307_ReadRaw`

```c
I2C_Status DS1307_ReadRaw(
    uint8_t raw_data[DS1307_TIME_REGISTER_COUNT]
);
```

Lee los siete registros de tiempo sin convertir.

Orden:

```text
raw_data[0] → segundos
raw_data[1] → minutos
raw_data[2] → horas
raw_data[3] → día de la semana
raw_data[4] → día del mes
raw_data[5] → mes
raw_data[6] → año
```

Ejemplo:

```c
uint8_t registros[7];

if (DS1307_ReadRaw(registros) == I2C_OK)
{
    /*
     * registros contienen los bytes BCD originales.
     */
}
```

---

### 11.3 `DS1307_ReadDateTime`

```c
I2C_Status DS1307_ReadDateTime(
    DS1307_DateTime *date_time
);
```

Lee, convierte y valida la fecha y hora.

Ejemplo:

```c
DS1307_DateTime fechaHora;
I2C_Status estado;

estado = DS1307_ReadDateTime(
    &fechaHora
);

if (estado == I2C_OK)
{
    if (fechaHora.data_valid != 0u)
    {
        /*
         * Los campos ya están en decimal.
         */
    }
}
```

La función puede devolver `I2C_OK` aunque los datos internos sean inválidos. En ese caso:

```c
fechaHora.data_valid == 0u
```

Esto diferencia un error del bus de un RTC que responde, pero contiene información no inicializada.

---

## 12. Validación de los datos

La librería valida:

- BCD de segundos.
- BCD de minutos.
- BCD de horas.
- BCD de fecha.
- BCD de mes.
- BCD de año.
- Segundos entre 0 y 59.
- Minutos entre 0 y 59.
- Horas entre 0 y 23 en modo de 24 horas.
- Horas entre 1 y 12 en modo de 12 horas.
- Día de la semana entre 1 y 7.
- Fecha entre 1 y 31.
- Mes entre 1 y 12.

La validación es básica. No comprueba todavía si una fecha específica existe, por ejemplo:

```text
31/02/2026
```

puede pasar la validación de rango general.

## 13. Ejemplo básico de lectura

```c
#include <xc.h>
#include <stdint.h>

#include "cabecera.h"
#include "I2C.h"
#include "DS1307.h"

void main(void)
{
    DS1307_DateTime fechaHora;
    I2C_Status estado;

    I2C1_Init();

    while (1)
    {
        estado = DS1307_ReadDateTime(
            &fechaHora
        );

        if (estado == I2C_OK)
        {
            if ((fechaHora.clock_running != 0u) &&
                (fechaHora.data_valid != 0u))
            {
                /*
                 * Datos disponibles:
                 *
                 * fechaHora.hours
                 * fechaHora.minutes
                 * fechaHora.seconds
                 * fechaHora.date
                 * fechaHora.month
                 * fechaHora.year
                 */
            }
        }

        __delay_ms(250);
    }
}
```

## 14. Ejemplo con LCD I2C

```c
#include <xc.h>
#include <stdint.h>

#include "cabecera.h"
#include "I2C.h"
#include "LCD_I2C.h"
#include "DS1307.h"

static void LCD_WriteTwoDigits(uint8_t value)
{
    LCD_I2C_WriteChar(
        (char)('0' + (value / 10u))
    );

    LCD_I2C_WriteChar(
        (char)('0' + (value % 10u))
    );
}

void main(void)
{
    DS1307_DateTime fechaHora;
    I2C_Status estado;

    I2C1_Init();

    LCD_I2C_SetAddress(PCF8574_7);
    LCD_I2C_Init();

    while (1)
    {
        estado = DS1307_ReadDateTime(
            &fechaHora
        );

        if ((estado == I2C_OK) &&
            (fechaHora.clock_running != 0u) &&
            (fechaHora.data_valid != 0u))
        {
            LCD_I2C_SetCursor(1u, 0u);

            LCD_WriteTwoDigits(fechaHora.hours);
            LCD_I2C_WriteChar(':');

            LCD_WriteTwoDigits(fechaHora.minutes);
            LCD_I2C_WriteChar(':');

            LCD_WriteTwoDigits(fechaHora.seconds);
        }
        else
        {
            LCD_I2C_SetCursor(1u, 0u);
            LCD_I2C_WriteString("RTC no disponible");
        }

        __delay_ms(250);
    }
}
```

## 15. Manejo de errores

### RTC desconectado

Normalmente:

```c
estado == I2C_ERROR_NACK
```

La aplicación puede mostrar un mensaje y volver a intentar más tarde.

### Oscilador detenido

```c
fechaHora.clock_running == 0u
```

La librería solamente informa el estado. Para iniciar el oscilador sería necesaria una función de escritura que coloque CH en cero.

### Datos inválidos

```c
fechaHora.data_valid == 0u
```

El DS1307 responde, pero sus registros contienen valores no válidos o no inicializados.

---

## 16. Día de la semana

El registro `weekday` contiene un valor de 1 a 7.

La relación entre número y nombre depende de cómo fue configurado el RTC.

Ejemplo de convención:

```text
1 → lunes
2 → martes
3 → miércoles
4 → jueves
5 → viernes
6 → sábado
7 → domingo
```

La librería no impone nombres para evitar asumir una convención distinta a la utilizada al configurar el dispositivo.

---

## 17. Año

El DS1307 almacena solamente los dos últimos dígitos.

La librería calcula:

```c
year = 2000u + valor;
```

Ejemplos:

```text
Registro 0x26 → 2026
Registro 0x99 → 2099
```

La librería está pensada para el intervalo 2000–2099.

---

## 18. Limitaciones

- Es una librería de solo lectura.
- No ajusta la fecha ni la hora.
- No puede iniciar el oscilador si CH está en 1.
- No configura SQW/OUT.
- No utiliza la RAM interna del DS1307.
- No valida el número real de días de cada mes.
- No comprueba años bisiestos en la validación.
- Utiliza dos transacciones para posicionar el puntero y leer.
- Depende de la recuperación de errores implementada en `I2C`.
- El DS1307 admite un máximo de 100 kHz en el bus.
